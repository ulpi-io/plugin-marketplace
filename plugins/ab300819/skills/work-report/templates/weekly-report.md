# 周报模板

为了更好地支撑月报、季报和年终总结的生成，建议按照以下格式编写周报。

## 邮件标题格式

```
YYYY.MM.DD[部门]-[团队]工作周报（姓名）
```

示例：`2025.08.08MIC后端研发部-商机组工作周报（张三）`

## 推荐格式

```markdown
1. 本月工作目标
   - 目标1：xxx
   - 目标2：xxx

2. 本周工作进展
   - 项目/模块 A
     - 进展描述，完成情况（X%）
     - 成果/数据
   - 项目/模块 B
     - 进展描述
     - 问题/风险（如有）

3. 本周小结（来自本周工作进展的总结）
   - 完成了xxx，达成了xxx目标
   - 遇到xxx问题，解决方案是xxx
   - 未完成xxx，原因是xxx，计划xxx

4. 感想和建议
   - （对工作、流程、协作等方面的思考）

5. 无效工作/工作损耗/浪费现象（非必填，有感而发即可）
   - （有一说一，拒绝内耗，聚焦价值）

6. 岗位提升：本周进展及心得体会（非必填，直叙胸臆即可）
   - （学习进展、技能提升、困难、心得）
```

## 行文要求

1. **结构层次清晰** - 使用标题、标号组织内容
2. **内容精炼** - 突出问题，表达核心观点，不求多
3. **格式紧凑** - 避免屏幕滚动，提升阅读效率

## 字段说明

| 字段 | 是否必填 | 说明 | 对报告生成的作用 |
| ---- | -------- | ---- | ---------------- |
| 本月工作目标 | 必填 | 本周计划达成的目标 | 目标与进展对照 |
| 本周工作进展 | 必填 | 各项工作的完成情况 | 月报/季报汇总 |
| 本周小结 | 必填 | 来自工作进展的总结（成果、问题、未完成） | 问题跟踪、经验积累 |
| 感想和建议 | 必填 | 对工作的思考和改进建议 | 个人成长素材 |
| 无效工作/浪费 | 可选 | 识别低效工作和浪费 | 效率改进 |
| 岗位提升 | 可选 | 学习进展和技能提升 | 年终总结成长部分 |

## 示例

```markdown
1. 本月工作目标
   - 完成 AISE 项目文档规范制定
   - 推进 Apifox 平台部署上线
   - MCP 协议开发达成 80% 进度

2. 本周工作进展
   - AISE 项目
     - 与姜平沟通确定文档框架（100%）
     - 产出《AISE 文档规范 v1.0》
   - Apifox 平台
     - 完成生产环境部署（100%）
     - 推广至后端、前端、测试团队
   - 代码追踪
     - MCP 协议开发完成（100%）
     - 前端集成中（80%）

3. 本周小结
   - 完成文档规范和平台部署两项核心目标
   - Agent 调试遇到上下文长度限制，已调整 prompt 策略优化
   - MCP 前端集成延期，需下周跟进联调

4. 感想和建议
   - 跨团队协作需提前对齐排期，避免等待
   - 建议将 Apifox 使用规范纳入新人培训

5. 无效工作/工作损耗/浪费现象
   - 某会议讨论内容与当前工作无关，可考虑按需参加

6. 岗位提升：本周进展及心得体会
   - 学习了 MCP 协议设计思想，对工具调用有了更深理解
```

## 周报内容对各类报告的支撑

| 周报字段 | 月报 | 季报 | 年终总结 |
| -------- | ---- | ---- | -------- |
| 工作目标 | 月度目标对齐 | 季度目标对齐 | 业绩达成 |
| 工作进展 | 汇总主要工作 | 汇总主要工作 | 业绩达成 |
| 本周小结 | 问题跟踪 | 问题解决 | 个人成长 |
| 感想和建议 | 改进建议 | 流程优化 | 成长计划 |
| 无效工作/浪费 | 效率分析 | 效率改进 | 辅助 |
| 岗位提升 | 技能积累 | 能力提升 | 个人成长 |
