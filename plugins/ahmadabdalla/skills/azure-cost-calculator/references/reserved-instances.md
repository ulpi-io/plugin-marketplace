| **Reservation term values are case-sensitive**      | The `reservationTerm` field in the API response (and any OData filters applied to it) accepts only exact values: `'1 Year'`, `'3 Years'`, or `'5 Years'` (note the space and capital letters). Always include the space and correct casing when matching or comparing these values.                        |
| **Not all services support Reserved Instances**     | Only certain Azure services offer RI pricing. Consumption-based services and many PaaS services typically do **not** support reservations. RI availability varies by service type.                                                                                                                                                  | Check the service reference file for RI examples. If none exist, assume RI is unavailable. The service reference file will document which services and SKUs support RI.                                                          |
| **Not all tiers within a service support RI**       | Within a service that supports reservations, only certain SKU tiers may be RI-eligible. Lower-tier or consumption-based SKUs within a service may return empty results with `-PriceType Reservation`.                                                                                                                               | Verify tier eligibility in the service reference file. If uncertain, query PAYG first to confirm the SKU exists, then retry with `-PriceType Reservation`. Service-specific RI restrictions are documented in each service file. |
| **RI queries may fail with meter name mismatches**  | Some services use different `meterName` or `productName` values for RI pricing vs PAYG. Copying the exact PAYG query and adding `-PriceType Reservation` may return nothing if the API uses different naming for reservations.                                                                                                      | If RI query returns empty: try removing `-ProductName` or `-SkuName` filters to broaden the search, or run the explore script with `PriceType: Reservation` to discover the correct names.                                       |
| **RI pricing varies by region significantly**       | Reserved Instance discounts are region-specific and may vary 5-15% across regions. Some regions may not offer RI for certain SKUs at all.                                                                                                                                                                                           | Always query RI pricing in the **same region** as the PAYG estimate. If comparing regions, run separate RI queries for each.                                                                                                     |
| **Multi-component services may have RI limits**     | Some services with multiple cost components (compute + storage, fixed + variable) may only support RI for certain components. Other components remain pay-as-you-go even with an RI.                                                                                                                                                | Check the service reference file for component-specific RI eligibility. You may need to run separate queries for RI-eligible and PAYG components, then sum both for total monthly cost.                                          |
| **RI `reservationTerm` field confirms term length** | The API response includes a `reservationTerm` field (e.g., `"1 Year"` or `"3 Years"`). This confirms the term you requested. If this field is missing or says `null`, the query returned PAYG pricing instead of RI â€” check your `-PriceType` and `-ReservationTerm` values.                                                        | Validate the `reservationTerm` field in the JSON response. If absent, the query did not match an RI offering.                                                                                                                    |
