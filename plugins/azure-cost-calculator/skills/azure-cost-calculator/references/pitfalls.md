# Pitfalls Reference — Known Traps, Discovery Workflow, Troubleshooting

## Known Pitfalls

Cross-cutting traps that apply across **all** services. These are API-level and script-level behaviours — not specific to any one resource type. For service-specific traps, see each service's reference file in `services/`.

### Cross-Platform Shell Traps

| Trap                                                  | Fix                                                                         |
| ----------------------------------------------------- | --------------------------------------------------------------------------- |
| **Bash/zsh strips OData quotes from `pwsh -Command`** | Use `pwsh -File script.ps1 ...` instead. Scripts handle quoting internally. |

### API Query Traps

| Trap                                                | Detail                                                                                                                                                                                                                                                                                                                                                                                                                                                          | Fix                                                                                                                                                                                  |
| --------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **`contains()` is case-sensitive**                  | The OData `contains()` operator used by the explore script is case-sensitive. A lowercase search term will return nothing if the actual value uses title-case or brand capitalisation.                                                                                                                                                                                                                                                                          | Use title-case or partial brand names. When in doubt, use a broader term.                                                                                                            |
| **`serviceName` mismatch with portal/docs names**   | The Azure Portal and documentation often use different display names than the Retail Prices API's `serviceName` field. These are different systems and there is no consistent mapping rule.                                                                                                                                                                                                                                                                     | Always verify `serviceName` via the explore script or the service reference files. Never guess from portal/docs names.                                                               |
| **"Azure" prefix in `productName` is inconsistent** | Some products include "Azure" in their `productName`, others do NOT. There is no consistent rule across the API — each service has its own naming convention.                                                                                                                                                                                                                                                                                                   | Always use the exact `productName` from the service reference file. If unsure, use the explore script to discover the exact value.                                                   |
| **Empty results ≠ API error**                       | The API returns HTTP 200 with an empty `Items` array for valid filters that match nothing. The script shows a warning but exits cleanly with no output.                                                                                                                                                                                                                                                                                                         | If no results: check case-sensitivity, check region (some services use `Global` — see below), try broader filters, use the explore script.                                           |
| **Some services use non-standard regions**          | Certain global or management-plane services list pricing under `armRegionName = 'Global'` or an empty string — not under any standard region. Querying any standard region (e.g., `eastus`, `australiaeast`) returns nothing silently. For `Global`-region services (e.g., Load Balancer, Private Link), pass `Region: Global` to the scripts. For empty-region services (e.g., Private DNS), the scripts cannot query them — use **direct API calls** instead. | Check the service reference file for region notes. If a service uses `Global` region, use `Region: Global`. If it uses an empty region, bypass the script and call the API directly. |
| **Not all services are in the Retail Prices API**   | Some services have **zero entries** in the API across all regions, search terms, and filter combinations.                                                                                                                                                                                                                                                                                                                                                       | Check [regions-and-currencies.md](regions-and-currencies.md) for the API-unavailable services list and use the manual fallback values provided there.                                |

### Result Interpretation Traps

| Trap                                             | Detail                                                                                                                                                                                                                                | Fix                                                                                                                                                                                    |
| ------------------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Unfiltered queries return mixed SKU variants** | Without `productName` or `skuName` filters, queries may return Spot, Low Priority, standard, and OS-variant (e.g. Linux/Windows) SKUs mixed together. The `summary.totalMonthlyCost` sums all of them, producing a misleading total.  | Always filter to the specific variant you need. Check `productName` and `skuName` in results to identify the correct row. Service reference files specify the exact filters.           |
| **Multi-meter resources need separate queries**  | Many resources have multiple cost components (e.g., fixed hourly + variable consumption, compute + storage). A single query without `MeterName` returns all meters mixed together, making the `summary.totalMonthlyCost` meaningless. | Run one query per meter with a specific `MeterName`. Service reference files show the exact queries needed for each component.                                                         |
| **Tiered pricing inflates summary totals**       | Some meters return multiple items with different `tierMinimumUnits` (e.g., storage GB tiers, per-key tiers). The `summary.totalMonthlyCost` sums all tiers, which is arithmetically meaningless.                                      | Calculate based on the expected quantity and the applicable tier bracket. Never trust `summary.totalMonthlyCost` for tiered meters. Service reference files document tier breakpoints. |

> For Reserved Instance pricing traps, see [reserved-instances.md](reserved-instances.md).

### Unit & Multiplier Traps

| Trap                                                       | Detail                                                                                                                                                                                                                                                                                    | Fix                                                                                                                                                                                                                  |
| ---------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **`unitOfMeasure` varies — don't assume hours**            | Most compute is `1/Hour` (× 730), but some meters use `1/Day` (× 30), `1/Month` (× 1), `1M` (per million), or `10K` (per 10,000). Applying the wrong multiplier gives wildly wrong results.                                                                                               | **Always** check `unitOfMeasure` in the API response or service reference before applying the monthly multiplier.                                                                                                    |
| **Script `MonthlyCost` may be wrong for non-hourly units** | The script calculates `MonthlyCost` using its default `HoursPerMonth` (730). For meters with `1/Day` or `1/Month` units, the script's auto-calculated `MonthlyCost` is **incorrect**. For `1/Day` meters it may show just the daily price; for `1/Month` it may show just the unit price. | For non-`1/Hour` units, **ignore** the script's `MonthlyCost` and manually calculate: `1/Day` → × 30, `1/Month` → × 1 (or × quantity), `10K` → × (operations ÷ 10,000), `1M` → × (operations ÷ 1,000,000).           |
| **Global-region prices return in USD only**                | Services priced under the `Global` region return prices in **USD** regardless of the `Currency` parameter or expectations. The script cannot query these services (region mismatch), and direct API calls return USD only.                                                                | Convert to the user's preferred currency using a known conversion factor if available. Note the conversion caveat to the user. See [regions-and-currencies.md](regions-and-currencies.md) for the derivation method. |

### Output Format Traps

| Trap                                           | Detail                                                                                                                                 | Fix                                                                                                  |
| ---------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------- |
| **`Write-Host` output is invisible to agents** | The `Summary` output format uses `Write-Host`, which writes to the host console stream — agents capturing stdout will see **nothing**. | Default to `OutputFormat: Json` (already the default). Only use `Summary` for human interactive use. |

## Discovery Workflow

Use this when the resource type is **not** in the service reference files:

1. **Search** — Use the explore script with `SearchTerm: <partial name>` and `OutputFormat: Json` to find available pricing entries
2. **Identify** — From the returned distinct combinations, pick the correct `serviceName`, `productName`, `skuName`, `meterName` values
3. **Estimate** — Use the pricing script with those exact values
4. **Caveat** — Note to the user that this resource type isn't in the reference files and the estimate may need manual verification

## Troubleshooting

If the script fails or returns no results:

- Check if `serviceName` value is correct (case-sensitive)
- Verify the region exists — some services use `Global` instead of a standard region
- Try broader filters first then narrow down
- Check the service reference file for known region or naming quirks
- Use the explore script as a discovery tool to find exact case-sensitive filter values for new resource types
