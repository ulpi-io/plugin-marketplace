# Status Indicator Reference

This document provides reference criteria for status indicators (🔴🟡🟢) used in value realization analysis. These criteria describe end users' cognitive state regarding value realization, serving as aids for analytical judgment. Each product, market, and end user group differs, and these should be applied within a specific context.

---

## Value Clarity

**What this examines**: Whether end users can articulate what they'll achieve.

**🟢 End users can articulate specific outcomes**

When asked "Why are you using this?":
- End users describe outcomes they'll achieve (e.g., "I can access my files from any device")
- End users' answers focus on results, not features
- End users can state in one sentence what they'll achieve

**🟡 End users understand the category but outcomes are unclear**

When asked "Why are you using this?":
- End users describe the product category or general purpose
- End users can explain what the product does, but not what they'll achieve
- End users' answers mix feature descriptions with outcome descriptions

**🔴 End users cannot articulate what they'll achieve**

When asked "Why are you using this?":
- End users describe features or technical capabilities (e.g., "because it has X")
- End users cannot state expected outcomes
- End users' answers focus on what the product can do, not what they'll achieve

## Value Timeline

**What this examines**: When value is realized, whether end users know when value is realized, and whether this matches the product's nature and end user context.

**🟢 Value timing matches product nature and end user context**

- Short-term value products: End users see results within minutes to hours
- Long-term value products: End users understand the timeline and commit to the journey
- Hybrid approach: End users working toward long-term goals also experience short-term value realization
- Value realization timing matches end users' expectations for this type of product

**🟡 Gap exists between value realization timing and end user expectations**

- Time when end users expect value realization differs from when value is actually realized
- Long-term value products: End users may disengage before value is realized
- Short-term value products: End users must wait longer than expected
- End users struggle to maintain engagement during the wait

**🔴 Value timing conflicts with end user expectations or product nature**

- Significant disconnect between when end users expect value realization and when value is actually realized
- End users don't know when value will be realized
- End users don't know how to maintain engagement until value is realized

## Value Perception

**What this examines**: Whether end users can see, feel, or demonstrate what they've achieved.

**🟢 End users can point to specific evidence of value realization**

- End users can identify concrete evidence of what they've achieved
- End users can perceive progress or outcomes of value realization
- End users can show others what they've achieved

**🟡 Value realization is partially visible to end users**

- Some aspects of value realization are visible to end users, others remain abstract
- End users can perceive value realization but need to interpret or infer it
- End users can perceive some evidence of value realization

**🔴 Value realization is invisible to end users**

- End users cannot identify concrete evidence of what they've achieved
- End users cannot perceive progress or outcomes of value realization
- End users cannot demonstrate what they've achieved

---

## Value Discovery

**What this examines**: How end users come to recognize the value they can achieve.

**🟢 End users recognize the value they can achieve through natural use**

- End users already know they want to achieve this type of value, or
- End users reach the "aha" moment within minutes of first use
- The value they can achieve becomes apparent to end users without explanation or education

**🟡 End users need guidance to recognize the value they can achieve**

- End users require guidance, tutorials, or education to recognize the value they can achieve
- Path to recognizing the value they can achieve is not immediately obvious to end users
- Some end users may fail to recognize the value they can achieve without guidance

**🔴 End users cannot find a clear path to discover the value they can achieve**

- End users require significant time or learning to recognize the value they can achieve
- End users cannot identify a value discovery mechanism
- End users likely stop using before recognizing the value they can achieve

---

## Usage Notes

**These are reference criteria**: Indicators describe end users' cognitive state regarding value realization. They aid judgment but are not rigid rules.

**Context determines judgment**: The same situation may correspond to different indicators in different contexts. Consider product type, market conditions, and end user characteristics.

**Analysis takes precedence over indicators**: When indicators conflict with detailed analysis, analysis takes precedence. Indicators aid assessment but don't replace analytical reasoning.

**May exhibit multiple characteristics**: A product may show characteristics of multiple statuses within a single dimension. The indicator represents comprehensive judgment based on complete analysis.

**Reference, not requirement**: These criteria serve as reference points. Detailed analysis explains the specific situation and reasoning process.
