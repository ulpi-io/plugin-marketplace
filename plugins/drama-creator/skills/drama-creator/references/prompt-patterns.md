# 竖屏短剧创作常用Prompt模式

本文件提供针对不同创作任务的常用prompt模式，帮助用户更有效地与skill交互。

## 任务模式Prompt模式

### 1. 宏观建构模式

**用途**: 设计整体故事大纲、规划长线剧情结构、布局核心付费卡点

**Prompt模板**:
```
基于以下创意，设计一个[集数]集的竖屏短剧大纲：
[创意描述/核心概念]

要求：
- 明确核心爽点和主打情绪
- 规划情绪曲线（压-放节奏）
- 设计关键付费卡点位置
- 创建人物小传
```

**示例**:
```
基于以下创意，设计一个10集的竖屏短剧大纲：
一个隐藏身份的战神总裁装穷体验生活，被女友嫌弃后分手，后来身份揭晓，前女友后悔不已。

要求：
- 明确核心爽点和主打情绪
- 规划情绪曲线（压-放节奏）
- 设计关键付费卡点位置
- 创建人物小传
```

---

### 2. 剧本创作模式

**用途**: 撰写完整的单集或多集剧本

**Prompt模板**:
```
创作第[N]集剧本，情节要点：
[具体情节描述]

要求：
- 本集是【压弹簧/放弹簧】集
- 遵循信息前置、动作可视化原则
- 设置开篇钩子和结尾悬念
- 字数控制在800字以内
```

**示例**:
```
创作第5集剧本，情节要点：
男主在餐厅被前女友和她的新男友羞辱，餐厅经理认出男主身份，当众揭晓男主是集团董事长。

要求：
- 本集是放弹簧集
- 遵循信息前置、动作可视化原则
- 设置开篇钩子和结尾悬念
- 字数控制在800字以内
```

---

### 3. 精准优化模式

**用途**: 对现有剧本进行情节修改、节奏调整、台词润色

**Prompt模板**:
```
优化以下剧本：
[原剧本内容]

优化方向：
- [具体问题1]
- [具体问题2]

要求：
- 说明修改原因
- 对比修改前后效果
```

**示例**:
```
优化以下剧本：
1. 咖啡厅 日 内
小李在喝咖啡，同事小张走过来。

小张：听说你要辞职了？

小李：是啊，我想换个环境。

小张：那你要去哪里？

小李：还没想好，可能去创业吧。

小张：加油，我支持你。

小李：谢谢。

优化方向：
- 对话平淡，缺少冲突
- 没有悬念，没有情绪峰值
- 信息量不足，没有吸引点

要求：
- 说明修改原因
- 对比修改前后效果
```

---

### 4. 创意发想模式

**用途**: 构思"打脸"场景、设计悬念钩子、头脑风暴人物小传

**Prompt模板**:
```
请帮我构思[数量]个[类型]的创意点子：

主题：[故事主题]
要求：
- [要求1]
- [要求2]
```

**示例**:
```
请帮我构思5个打脸场景的创意点子：

主题：隐藏大佬身份被揭晓
要求：
- 每个场景有不同的揭晓方式
- 震撼程度递增
- 适合竖屏短剧的表现形式
```

## 按创作阶段分类的Prompt模式

### 前期策划阶段

#### 情绪价值分析Prompt
```
分析以下故事创意的核心情绪价值：

[创意描述]

请分析：
1. 核心情绪是什么（爽/甜/虐/悬/共鸣/温暖）？
2. 情绪触发点有哪些？
3. 如何设计情绪曲线？
```

#### 人物设计Prompt
```
设计一个[类型]的主角人设：

故事背景：[背景描述]

要求：
- 设计3个核心标签
- 标签要极致化
- 明确人设容器功能（承载什么愿望/情绪/幻想）
- 设计反差（表面vs真实）
```

---

### 中期创作阶段

#### 开篇钩子设计Prompt
```
设计一个[类型]的开篇钩子（前3秒）：

故事背景：[背景描述]

要求：
- 第一秒：视觉冲击
- 第二秒：台词冲击
- 第三秒：抛出悬念
```

#### 冲突场景设计Prompt
```
设计一个冲突场景：

场景类型：[误会/对峙/羞辱/危机]
参与人物：[人物列表]
情绪目标：[压弹簧/放弹簧]

要求：
- 动作可视化
- 信息前置
- 情绪强烈
```

#### 身份揭晓场景设计Prompt
```
设计一个身份揭晓场景：

隐藏身份：[具体身份]
揭晓场景：[地点、场合]
在场人物：[人物列表]

要求：
- 分层次揭晓（铺垫-暗示-揭晓）
- 多视角反应（主角/配角/反派）
- 震撼效果最大化
```

---

### 后期优化阶段

#### 节奏检查Prompt
```
检查以下剧本的节奏问题：

[剧本内容]

请分析：
1. 有无拖沓情节？
2. 钩子是否吸引人？
3. 情绪功能是否明确？
4. 如何优化节奏？
```

#### 台词优化Prompt
```
优化以下场景的台词：

[原剧本片段]

优化目标：
- 信息前置
- 删除废话
- 符合人物性格
- 强化冲突或情绪
```

#### 悬念设计Prompt
```
为以下场景设计结尾悬念：

[场景描述]

要求：
- 悬念类型：[身份/真相/危机/反转]
- 必须让观众想看下一集
- 与主线剧情相关
```

## 按短剧类型分类的Prompt模式

### 都市逆袭类
```
创作一个都市逆袭类剧本：

核心设定：[主角身份+逆袭原因]

要求：
- 前3集持续压弹簧（积蓄负面情绪）
- 第4-5集开始放弹簧（身份揭晓）
- 设计至少3个打脸场景
- 反派要嚣张到让观众想打他
```

### 甜宠爱情类
```
创作一个甜宠爱情类剧本：

核心设定：[男女主关系]

要求：
- 男主宠溺要极致
- 设计误会-解除的循环
- 甜-虐-甜的情绪曲线
- 每集都有小甜点
```

### 悬疑推理类
```
创作一个悬疑推理类剧本：

核心谜题：[谜题描述]

要求：
- 每集都有新线索
- 真相要隐藏到最后一集
- 设置假线索误导
- 危机感贯穿始终
```

## 快速迭代优化Prompt

### 迭代优化模式1：直接指出问题
```
这段剧本有什么问题？请直接指出：

[剧本内容]

请从以下维度分析：
- 情绪功能
- 节奏问题
- 台词问题
- 格式规范
```

### 迭代优化模式2：针对性修改
```
请针对[具体问题]修改以下剧本：

[剧本内容]

修改要求：
- 保持其他部分不变
- 说明修改原因
```

### 迭代优化模式3：对比版本
```
请给出两个版本的剧本：

原场景：[场景描述]

版本1：保守修改（小幅优化）
版本2：激进修改（完全重构）

请对比说明两个版本的优缺点。
```

## 特定场景创作Prompt

### 场景1：分手场景
```
创作一个分手场景：

分手原因：[原因描述]
分手方式：[和平/激烈/单方面]

要求：
- 情绪要强烈
- 台词要精炼
- 为后续剧情埋下伏笔
```

### 场景2：重逢场景
```
创作一个重逢场景：

分离时长：[时长]
分离原因：[原因]
重逢地点：[地点]

要求：
- 时光流逝感
- 情感涌动
- 视觉冲击（如特写、慢镜头）
```

### 场景3：危机救援场景
```
创作一个危机救援场景：

危机类型：[绑架/威胁/危险]
救援者：[人物]
救援方式：[武力/智谋/身份]

要求：
- 危机要紧迫
- 救援要及时
- 释放要充分
```

## 完整项目创作Prompt流程

### 完整流程示例
```
阶段1：宏观建构
"基于创意：[创意]，设计10集短剧大纲"

阶段2：人物设计
"为主角[名字]设计详细人设"

阶段3：单集创作
"创作第1集剧本：[情节要点]"
"创作第2集剧本：[情节要点]"
...

阶段4：优化调整
"检查并优化第1-3集的节奏和情绪曲线"
"为第5集设计更强的结尾悬念"

阶段5：整体检查
"整体检查10集剧本的连贯性和情绪曲线"
```

## Prompt编写技巧

### 技巧1：明确情绪功能
在prompt中明确指出本场景是"压弹簧"还是"放弹簧"，这能帮助skill更好地把握情绪方向。

### 技巧2：提供具体情节
描述情节时越具体越好，避免模糊的表述。
- ❌ "写一个打脸场景"
- ✅ "写一个在餐厅打脸的场景，前女友羞辱男主，经理揭晓男主身份"

### 技巧3：设置约束条件
明确约束条件能获得更精准的输出：
- 字数限制："800字以内"
- 时长限制："2分钟以内"
- 场景限制："不超过3个场景"

### 技巧4：指明创作风格
告知skill期望的风格：
- "台词要犀利"
- "节奏要紧凑"
- "反转要出人意料"

### 技巧5：利用迭代
如果第一次输出不满意，继续迭代：
- "这个版本太温和了，请加强冲突"
- "台词还有点废话，请进一步精简"
- "悬念不够强，请设计一个更强的结尾钩子"
