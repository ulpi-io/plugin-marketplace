# Wireframing Skill

A comprehensive Claude Code skill for creating low-fidelity and high-fidelity wireframes, user flows, information architecture, and prototyping.

## Overview

This skill provides detailed knowledge and best practices for wireframing in UX/UI design. Whether you're sketching initial concepts or creating detailed specifications for developers, this skill covers the entire spectrum of wireframing techniques and methodologies.

## What You'll Learn

### Core Wireframing Concepts
- **Fidelity Levels**: Understanding when to use low, mid, or high-fidelity wireframes
- **Visual Language**: Consistent representation of UI elements and interactions
- **Layout Principles**: Grid systems, spacing, and information hierarchy
- **Component Libraries**: Reusable patterns and design systems

### Information Architecture
- **Sitemaps**: Hierarchical structure of websites and applications
- **Content Hierarchy**: Organizing information by importance and relationships
- **Card Sorting**: Understanding user mental models
- **Navigation Systems**: Primary, secondary, and utility navigation patterns

### User Flows and Journeys
- **Task Flows**: Step-by-step paths for specific user goals
- **User Journeys**: Emotional and experiential mapping across touchpoints
- **Flowcharts**: Visualizing logic and decision points
- **Wireflows**: Combining wireframes with flow diagrams

### Professional Tools
- **Digital Tools**: Figma, Sketch, Adobe XD, Balsamiq, Whimsical, Axure RP
- **Diagramming Tools**: Miro, Lucidchart, OmniGraffle
- **Paper Prototyping**: Quick sketching and brainstorming techniques
- **Code-Based Wireframes**: HTML/CSS prototypes

### Design Process
- **Iteration Cycles**: From concept to high-fidelity
- **Feedback Loops**: Design critiques, stakeholder reviews, user testing
- **Progressive Refinement**: When to increase fidelity
- **Collaboration**: Working with developers, stakeholders, and users

## Getting Started

### For Beginners

If you're new to wireframing:

1. **Start with Paper**: Sketch rough ideas with pen and paper
2. **Use Low-Fidelity Tools**: Try Balsamiq or Whimsical for simple digital wireframes
3. **Focus on Structure**: Don't worry about visual design yet
4. **Test Early**: Get feedback on rough concepts before adding detail
5. **Study Examples**: Review the 15+ examples in this skill

### For Intermediate Designers

If you have some wireframing experience:

1. **Master Fidelity Progression**: Learn when to increase detail
2. **Improve Annotations**: Document interactions more thoroughly
3. **Explore Tools**: Try Figma or Adobe XD for more advanced work
4. **Create Systems**: Build reusable component libraries
5. **Focus on Flows**: Document complete user journeys

### For Advanced Practitioners

If you're an experienced designer:

1. **Optimize Process**: Streamline your wireframing workflow
2. **Advanced Prototyping**: Create interactive, high-fidelity prototypes
3. **Design Systems**: Build comprehensive pattern libraries
4. **Teach Others**: Help junior designers improve their skills
5. **Tool Mastery**: Explore advanced features in Axure or Figma

## Workflow Integration

### Typical Design Process

1. **Discovery & Research**
   - Understand user needs and business requirements
   - Review existing solutions and competitors
   - Define success metrics

2. **Information Architecture**
   - Create sitemaps
   - Define content hierarchy
   - Map user flows

3. **Low-Fidelity Wireframes**
   - Sketch multiple concepts
   - Explore different approaches
   - Get quick feedback

4. **User Testing**
   - Test core concepts with users
   - Validate information architecture
   - Identify major usability issues

5. **Mid-Fidelity Wireframes**
   - Refine structure based on feedback
   - Add more detail
   - Document interactions

6. **Stakeholder Review**
   - Present to business stakeholders
   - Confirm alignment with requirements
   - Get buy-in for direction

7. **High-Fidelity Wireframes**
   - Create detailed specifications
   - Document all states and edge cases
   - Prepare for developer handoff

8. **Interactive Prototypes**
   - Build clickable prototypes
   - Conduct usability testing
   - Demonstrate to stakeholders

9. **Visual Design**
   - Apply brand and visual design
   - Create high-fidelity mockups
   - Design system implementation

10. **Developer Handoff**
    - Provide comprehensive specifications
    - Support during development
    - Review implementation

## Common Use Cases

### Creating a New Product

**Challenge**: Designing a new application from scratch

**Approach**:
1. Sketch multiple concepts on paper
2. Create low-fi wireframes for top concepts
3. Build user flow diagrams
4. Test basic concepts with users
5. Iterate to mid-fi with feedback
6. Create high-fi specifications for development

### Redesigning an Existing Interface

**Challenge**: Improving an existing product's UX

**Approach**:
1. Audit current experience and identify pain points
2. Sketch improved solutions
3. Create wireframes showing before/after
4. Test new design with users
5. Iterate based on feedback
6. Provide detailed specifications for implementation

### Adding a New Feature

**Challenge**: Integrating new functionality into existing product

**Approach**:
1. Map where feature fits in existing flows
2. Wireframe integration points
3. Create detailed wireframes for new screens
4. Document interactions with existing features
5. Test to ensure seamless integration

### Stakeholder Alignment

**Challenge**: Getting agreement on design direction

**Approach**:
1. Create multiple low-fi concepts
2. Present options with pros/cons
3. Facilitate discussion and decision-making
4. Refine chosen direction
5. Document decisions and rationale

### Developer Handoff

**Challenge**: Clearly communicating design to developers

**Approach**:
1. Create high-fidelity wireframes
2. Annotate all interactions and states
3. Document edge cases and error handling
4. Provide specifications for spacing and layout
5. Be available for questions during development

## Best Practices Summary

### Do's

- **Start Low-Fidelity**: Begin with rough sketches and simple wireframes
- **Iterate Often**: Make changes quickly based on feedback
- **Test Early**: Validate concepts before detailed work
- **Document Clearly**: Annotate interactions and specifications
- **Think Mobile-First**: Consider responsive design from the start
- **Focus on Users**: Prioritize user needs over stakeholder preferences
- **Use Consistent Patterns**: Maintain design system principles
- **Plan for Edge Cases**: Consider errors, empty states, and unusual scenarios
- **Collaborate**: Involve stakeholders, developers, and users throughout
- **Version Control**: Track changes and maintain design history

### Don'ts

- **Don't Add Visual Design Too Early**: Focus on structure first
- **Don't Skip User Testing**: Validate assumptions with real users
- **Don't Work in Isolation**: Engage your team throughout the process
- **Don't Ignore Accessibility**: Consider all users from the beginning
- **Don't Over-Design**: Match fidelity to your stage in the process
- **Don't Forget Mobile**: Consider all screen sizes and devices
- **Don't Skip Annotations**: Document your thinking and specifications
- **Don't Get Attached**: Be willing to change or discard ideas
- **Don't Ignore Edge Cases**: Plan for errors and unusual scenarios
- **Don't Assume**: Test and validate your design decisions

## Accessibility Considerations

Wireframes should incorporate accessibility from the start:

### Structure
- Logical heading hierarchy (H1, H2, H3)
- Semantic HTML structure
- Landmark regions defined

### Navigation
- Keyboard navigation support
- Focus indicators planned
- Skip links for long pages
- Logical tab order

### Content
- Alternative text for images noted
- Captions for video/audio planned
- Color not sole indicator
- Sufficient contrast considered

### Interaction
- Touch targets minimum 44x44px
- Error messages clearly associated
- Form labels properly linked
- ARIA labels documented where needed
