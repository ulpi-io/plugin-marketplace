## Edge Cases

- Meeting notes from project sync -> Project folder
- Meeting notes from recurring operations review -> Area folder
- Research for active launch -> Project first, then extract durable notes to Resources
- Quarterly planning -> Create project instance each quarter (for example, "Q2 2026 Planning")

## Common Misclassifications (10)

1. ❌ "Marketing" as a Project -> ✅ Area (`Marketing Operations`)
2. ❌ "Learning Spanish" as a Project -> ✅ Area (ongoing practice)
3. ❌ "Research for active migration" in Resources -> ✅ Project (`Billing Migration`)
4. ❌ All meeting notes in Resources -> ✅ Route by meeting purpose (Project or Area)
5. ❌ Keep completed tax filing in Projects -> ✅ Move to Archive
6. ❌ Create folder for one-off note -> ✅ Keep under broader existing folder
7. ❌ Quarterly planning as one recurring item -> ✅ Separate project each cycle
8. ❌ Put routine health habits in Projects -> ✅ Area (`Health`)
9. ❌ Archive reusable playbook only inside closed project -> ✅ Extract to Resources, archive project
10. ❌ Put active certification prep in Resources -> ✅ Project until completed
